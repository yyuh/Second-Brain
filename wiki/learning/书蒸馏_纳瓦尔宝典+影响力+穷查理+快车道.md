## 第一本：《纳瓦尔宝典》— Naval Ravikant

### 一句话
财富是你睡觉时也在赚钱的能力，幸福是一种技能，两者都可习得。

### 核心骨架（Naval 的四条因果链）

```
【财富】
    ┌─ 专长（不能培训的，天生热爱+擅长的）
    ├─ 杠杆（代码/媒体/资本——AI是你的代码型杠杆）
    ├─ 复利（不仅是钱，信任和知识也复利）
    └─ 拥资产（ownership）≠ 出卖时间（打工）
    └→ 退休 = 不再为明天焦虑

【幸福】
    ┌─ 欲望是你和别人比较的契约
    ├─ 接受当下 + 改变你能改变的
    ├─ 冥想=观察你的思维模式
    └─ 幸福是默认状态，焦虑是后天习得的
```

### 与泰明当前处境的交点

| 你的现状 | Naval 打在哪 |
|----------|------------|
| AI当施工队，自己负责产品判断 | ✅ 代码杠杆（最高杠杆率） |
| 财富自由棋做引流产品 | ✅ 拥资产，不卖时间 |
| 定投+期权学习 | ✅ 睡后收入 = 财富的本质 |
| 家族建材年利润60w（辅助角色） | ✅ 专长（你对西藏市场的判断力） |

### 可摄入智者系统的思维模型

**模型A：杠杆三阶论**
- **核心：** 劳动力杠杆（管人）< 资本杠杆（管钱）< 代码/媒体杠杆（0边际成本复制）
- **推论：** AI时代，代码杠杆的门槛降到最低，这是普通人最后一次追平差距的机会
- **反方：** 代码杠杆虽然门槛降低，但产品判断力和用户理解仍然是稀缺品

**模型B：专长=热爱+独特经验**
- **核心：** 专长不能被培训或外包，它是你天生热爱之事与你独特生活经历的交集
- **推论：** 不必焦虑"我不够专业"，你的家族建材经验+物联网背景+AI能力就是独一无二的三角
- **反方：** "热爱"可能被用来合理逃避困难工作

**模型C：退休≠不工作**
- **核心：** 退休不是60岁不干活，而是「明天不做今天也能活」。做热爱的事不求回报。
- **推论：** 追求的不是大赚一笔然后躺平，而是拥有不焦虑的选择权

## 第二本：《影响力》— Robert Cialdini

### 一句话
人有6个自动顺从开关，谁先识别谁掌握谈判/营销的主动权。

### 核心骨架（6大原则）

```
1. 互惠（Reciprocity）
   → 你先给，对方自然想还。先送价值再谈生意。
   
2. 承诺一致性（Commitment & Consistency）
   → 人一旦公开承诺某件事，会不自觉保持一致。从小承诺引导到大承诺。

3. 社会认同（Social Proof）
   → 人做决定时看"别人怎么做"。你卖的不是产品，是大家都在用的安全感。

4. 喜好（Liking）
   → 人更容易对喜欢的人说Yes。相似性、赞美、熟悉感。

5. 权威（Authority）
   → 人对权威有服从惯性。专业形象=信任开关。

6. 稀缺（Scarcity）
   → 得到的怕失去，没得到的怕错过。限量和期限创造价值感。
```

### 与泰明当前处境的交点

| 你的场景 | Cialdini 怎么用 |
|----------|--------------|
| 财富自由棋文案转化 | 社会认同（已有X人免费试玩）+ 稀缺（限时免费） |
| 建材客户信任 | 权威（专业建议）+ 互惠（先给免费咨询服务） |
| 闲鱼/社区推广 | 承诺一致性（先让用户"关注"这个动作起承诺） |
| AI产品定价 | 稀缺（前20单9.9→29.9，你已经直觉在用了） |

### 可摄入智者系统的思维模型

**模型D：自动顺从启动器**
- **核心：** 人类大脑有大量自动行为模式，特定触发词/动作能绕过理性直接启动顺从
- **推论：** 营销不是说服用户，是找到对方的自动开关并按下去
- **反方：** 滥用导致信任崩塌，一次欺骗终身拉黑

**模型E：承诺阶梯论**
- **核心：** 让用户先做一个小而轻易的承诺，会自然引导他做更大的承诺
- **推论：** 推广产品的第一步不是让人买，是让人关注、点赞、试玩——然后买是自然动作
- **反方：** 用户如果意识到自己被"引导"，会产生背叛感

**模型F：对比原理**
- **核心：** 先给贵的选项，便宜的自然看起来更划算。人不是绝对判断，是比较判断。
- **推论：** 定价时先放高端方案，用户会觉得中档方案"太值了"

## 第三本：《穷查理宝典》— Charlie Munger

### 一句话
你需要100个跨学科的思维模型构成的格栅，才能做出不愚蠢的决定。

### 核心骨架（Munger的模型网络）

```
【投资决策的前提】
    不做什么 > 做什么
    能力圈 → 只在你懂的东西里做决定
    反过来想 → 如果不知道什么是好的，先知道什么会毁了你

【关键模型提取（与你的76条互补）】
    1. 复利 —— 你已有（m003）
    2. 机会成本 —— 你已有（m038）
    3. 确认偏误 —— 你已有相似概念
    4. 激励机制 —— 人会被激励驱动，设计机制而不是教育人 ★★★ 新补充
    5. 双轨分析 —— 理性分析+潜意识分析同时做 ★★★ 新补充
    6. 反证法 —— 证明一个东西是假，比证明它为真更容易 ★★★ 新补充
    7. 能力圈 —— 知道边界比知道里面更重要 ★★★ 新补充

【Munger的决策清单（双重检查）】
    □ 我够资格判断这件事吗？（能力圈检查）
    □ 相反的证据我认真看了吗？（反证法）
    □ 这个问题涉及什么激励？（激励机制分析）
    □ 我的潜意识在驱动我做这个决定吗？（双轨分析）
    □ 如果10年后回看，这个决定还正确吗？（复利检查）
```

### 与泰明当前处境的交点

| 你的现状 | Munger 打在哪 |
|----------|------------|
| 思维模型已76条 | 但缺少决策检查清单（从框架→真正用起来的一步） |
| 做产品判断 | 需要能力圈检查：我该自己做还是外包哪些部分 |
| 投资期权学习 | 反过来想：不亏比赚更重要，避免愚蠢比追求聪明更有效 |
| 家庭建材经营 | 激励机制：父亲和你的激励是否一致？客户和销售呢？ |

### 可摄入智者系统的思维模型

**模型G：能力圈法则**
- **核心：** 知道自己不懂什么比知道懂什么更重要。在圈外做的决定都是赌。
- **推论：** 做产品坚决不做不懂的领域，AI只是施工队，产品判断力才是护城河
- **反方：** 能力圈太窄会错过交叉领域的创新

**模型H：激励机制超级力量**
- **核心：** 人做事的根本驱动力是激励，改变行为最有效的方式不是教育而是重设激励
- **推论：** 家族建材生意中，你想让父亲用新的管理方法？先给激励，不是讲道理
- **反方：** 激励设计不当会导致游戏规则层面的作弊

**模型I：双向思维（反证法优先）**
- **核心：** 做决定前先想"什么会让这个决定变得非常糟糕"，然后避免它
- **推论：** 做产品先想"什么会杀死亡国富自由棋"（抄袭？审核？留存差？）→ 先堵死这些
- **反方：** 过度使用会让人变得保守不敢行动

## 第四本：《百万富翁快车道》— MJ DeMarco

### 一句话
财富的三个车道——人行道（月光族）、慢车道（打工存钱等退休）、快车道（创业拥资产）——选对车道比努力重要一百倍。

### 核心骨架（DeMarco的财富地图）

```
【人行道】— 靠债务维持今朝有酒今朝醉
    → 终局：贫困

【慢车道】— 打工 → 存钱 → 投401k → 65岁退休
    → 终局：勉强温饱（如果算上通胀）
    → 公式：财富 = 月薪 × 储蓄率 × 时间（太慢！）

【快车道】— 创业 → 拥资产 → 系统化 → 影响百万人
    → 公式：财富 = 净利润 + 资产价值
    → 5条快车道法则：
        需求法则（解决真实需求，不是自嗨）
        进入法则（进入门槛越高越好，越多人做越不值钱）
        控制法则（全盘控盘，不做受制于人的生意）
        规模法则（影响越大，上限越高）
        时间法则（时间与收入脱钩——不卖时间）
```

### 与泰明当前处境的交点

| 你的现状 | DeMarco 打在哪 |
|----------|-------------|
| 月薪4000保安全感（慢车道） | ✅ 你在慢车道上但认知已到快车道 |
| AI做产品 | ✅ 快车道+时间法则：代码杠杆让时间与收入脱钩 |
| 财富自由棋小程序 | ✅ 规模法则：小程序免费传播有规模潜力 |
| 家族建材（年利润60w） | ✅ 控制法则：你对这块有完全的判断权 |
| 定投/期权学习 | 快车道+投资：先快速道搞到本金，再投资滚雪球 |

### DeMarco vs 你学的那套期权策略

| | 快车道（DeMarco） | 期权卖方（你的策略） |
|---|---|---|
| 阶段定位 | 35岁前 | 35岁后本金够大 |
| 核心 | 创业造资产 | 资金效率管理 |
| 杠杆 | 产品/媒体 | 保证金/期权 |
| 风险 | 产品失败 | 黑天鹅 |
| **正确顺序** | **先快车道创第一桶金** | **再有本钱玩期权游戏** |

### 可摄入智者系统的思维模型

**模型J：车道选择决定论**
- **核心：** 财富量级的差异来自你选择在哪条车道上跑，而不是你跑得多努力
- **推论：** 月薪4000是慢车道，但可以作为安全垫让你心无旁骛走快车道
- **反方：** 快车道的失败率远高于慢车道，不是每个人都适合

**模型K：需求≠自嗨**
- **核心：** 赚钱的唯一路径是解决真实需求。你的产品必须先被需要，再被喜欢。
- **推论：** 财富自由棋引流，真正卖的是什么？（后续产品/知识付费/工具？）需求要先验证
- **反方：** 有些需求是被创造出来的（iPhone），但那是少数天才的路径

**模型L：进入门槛=护城河**
- **核心：** 越多人能做的事越不值钱。快车道的核心是做到别人做不到的事。
- **推论：** 你做AI产品的护城河是什么？不是代码（谁都能写），是你对建材/投资/特定人群的理解

**模型M：慢车道作为安全垫**
- **核心：** 慢车道本身不是终点，但它提供的社保和稳定现金流可以让你在快车道上更敢搏
- **推论：** 你的月薪4000是宝贵的——它让你可以不在乎短期赚钱，押注长期产品
- **反方：** 人容易沉溺于慢车道的舒适区，误以为安全=安逸
