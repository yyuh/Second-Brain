# 智者架构设计 v0.1

**日期：** 2026-06-16
**目标：** 支持双模式运转（全融合智者模式 + 单人角色切换模式）

## 产品理念

智者不是另一个AI聊天助手，是一个**认知切换器**。用户打开智者，选择了体验模式后，面对的是一颗融合了多位优秀头脑的复合大脑。

## 详细设计

### 1. 知识源层（Knowledge Source Layer）

**唯一规范源：** `second-brain/wiki/insights/思维模型.md`（47条模型）
**人物技能目录：** `second-brain/skills/` 下的每个子目录

技能目录命名规范：
```
second-brain/skills/
├── ludamo/           # 鲁大魔 (dir命名 = 用户触发词)
│   └── SKILL.md
├── sun-libo-liang/   # 太阳李博良
│   └── SKILL.md
├── jack-uncle/       # Jack叔叔
│   └── SKILL.md
├── .../
```

**知识索引文件（自动生成）：**
```json
// core/characters.json
{
  "characters": [
    {"trigger": "鲁大魔", "dir": "ludamo", "title": "鲁大魔 · 认知升维教练"},
    {"trigger": "李博良", "dir": "sun-libo-liang", "title": "太阳李博良 · 政治哲学"},
    {"trigger": "JACK", "dir": "jack-uncle", "title": "Jack叔叔 · 人生RPG导师"}
  ]
}
```

### 2. 模式选择器（Mode Router）

**智者模式（默认）：**
- 将47条模型全量注入 AI System Prompt
- 不附加任何单人SKILL.md
- 回答风格：融合、中立、多维视角
- 适合"我想学习/思考一个问题"

**单人模式：**
- 加载指定角色的 SKILL.md 作为 System Prompt 的一部分
- 47条模型依然存在但被角色过滤器压制
- 回答风格：模仿该角色的思考方式和表达DNA
- 适合"我想体验跟XX聊天的感觉"

### 3. 融合引擎（Fusion Engine）

智者模式下的回答生成逻辑：

```
When user asks a question:
  1. Parse question - 识别领域（投资/人生/创业/哲学）
  2. Retrieval - 从47条模型中召回相关度Top 5
  3. Contextualize - 将召回的模型融入回答骨架
  4. Voice - 用智者统一口吻输出（非某个博主的语气）
  5. Tagging - 输出完后可显示"关联模型：[m003] [m022] [m033]"
     （卡片化展示，不内嵌在回答中）
```

### 4. 角色过滤器（Persona Filter）

单人模式下的回答生成逻辑：

```
When user asks a question (mode=某角色):
  1. Load SKILL.md for that character → 作为 persona frame
  2. 47条模型在后备层（如果角色自己的知识不够）
  3. 表达DNA从SKILL.md中的「表达DNA」字段提取
  4. 回答风格完全服从角色设定
  5. 保持角色贯穿整个对话，直到用户切换
```

### 5. 上下文管理

```
用户说"开启鲁大魔" → session.mode = "single", session.activeChar = "ludamo"
用户说"切回智者"  → session.mode = "wiseman", session.activeChar = null
用户说"换李博良"  → session.mode = "single", session.activeChar = "sun-libo-liang"
```
